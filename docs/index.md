# 図解でわかるLinux基礎

図解とイラストで楽しく学ぶLinux入門書へようこそ。  
この本では、複雑になりがちなLinuxの概念を図を使ってかみくだき、初心者でも読み進めやすい形で解説します。

読み終わるころには、基本的なコマンドやOSの仕組みを、自分の言葉で説明しながら実行できるようになることを目標とします。

## この本を読む前に

### 想定読者

- Linuxやコマンドラインに興味があるが、本格的に触ったことはない方
- 新卒エンジニアやインターン、インフラ・バックエンドにチャレンジしたい方
- 既にLinuxを使っているが、基礎を図で整理し直したい方

### 前提知識

- マウスやキーボードを使った基本的なPC操作に慣れていること
- WindowsやmacOSなど、一般的なデスクトップOSを日常的に使ったことがあること

プログラミング経験やLinuxの予備知識がなくても読み進められる想定です。

## 学習成果

- コマンドラインでファイルやディレクトリを操作し、簡単な日常作業をLinux上でこなせるようになる。
- プロセスやメモリ、ディスク使用状況など、基本的なシステム状態を確認し、簡単な異常の兆候を自分で見つけられるようになる。
- ファイル権限やユーザー・グループの仕組みを、図を用いて他人に説明できるレベルで理解できる。
- 初心者がよく遭遇するエラーの原因を推測し、エラーメッセージや基本コマンドを使って対処できるようになる。

## 読み方ガイド

- Linux をほとんど触ったことがない読者は、第0章でコンピュータやOSの全体像を掴んだあと、第1章の必須コマンドを一通り試してみるところから始めるとよい。
- すでにコマンドに少し触れたことがあり、権限やユーザー管理に苦手意識がある読者は、第1章を軽く復習したうえで、第2章を丁寧に読む読み方も有効である。
- 実務でLinuxを使っており、トラブル対応やテキスト処理に課題を感じている読者は、第3章と第4章を先に読み、必要に応じて基礎章に戻るパターンも選べる。
- シェルスクリプトに興味がある読者は、第1〜3章で基礎を押さえた後、第5章の演習を通じて「日常のちょっとした自動化」を体験すると、以降の学習の動機づけになりやすい。

## 所要時間
- 通読: 約2〜2.5時間（本文量ベース概算。400〜600文字/分換算）
- 章末の演習を実施する場合は、追加で時間が必要です（環境/習熟度で変動）。

## 読み方ガイド

- まずは第0章で「OS/サーバー/ハードウェア」の全体像をつかみましょう。
- 第1章のコマンドは、手元で実行して動きを確認しましょう（最初は「暗記」より「体験」を優先）。
- 権限でつまずいたら第2章に戻りましょう。テキスト処理やインストールでつまずいたら第3章に戻りましょう。
- エラー対応は第4章を「辞書」として参照しましょう。
- 最後に第5章で自動化（スクリプト/cron）に進みましょう。

## この本の特徴

- **図解中心**：複雑な概念も分かりやすい図解で説明
- **実践的**：実際に使える知識とスキルを習得
- **段階的学習**：基礎から応用まで順序立てて学習
- **トラブル対応**：よくある問題の解決方法も収録
